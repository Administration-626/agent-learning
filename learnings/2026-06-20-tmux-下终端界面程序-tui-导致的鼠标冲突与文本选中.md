# tmux 下终端界面程序（TUI）导致的鼠标冲突与文本选中

Date: 2026-06-20
Agent: Gemini CLI

## Background
在开启了鼠标支持（`set -g mouse on`）的 tmux 环境中运行交互式的命令行程序（TUI，如部分 AI Agent 客户端）时，用户经常会遇到无法使用鼠标拖拽来高亮和复制输出文本的问题。

## Context / Problem
- tmux 的 `set -g mouse on` 选项会接管整个终端的鼠标控制权。
- 现代 TUI 程序往往也会请求捕获鼠标（为了实现点击按钮或滚轮翻页等功能）。
- **复制冲突**：当这两者结合时，终端模拟器会将鼠标点击事件作为“内部交互”传递给底层程序，从而关闭了系统原生的纯文本“框选”功能。此时鼠标点击会被“吃掉”，表现为无法选中任何文字。
- **历史浏览陷阱 (Copy Mode Trap)**：如果在 tmux 中使用鼠标滚轮向上滚动查看历史，tmux 会自动进入「复制/浏览模式 (Copy Mode)」（右上角出现行号）。此时屏幕冻结，键盘输入被 tmux 整体接管。底层的 TUI 程序根本收不到 `End`、`PageDown` 等任何按键输入，导致用户以为卡死，无法回到最底下的输入行。

## Solution / Learnings

面对上述多层级劫持，有以下解决方式：

### 1. 解决文本无法选中的问题：使用修饰键强制绕过（推荐）
绝大多数现代终端模拟器都提供了一个“安全后门”，通过按住修饰键，可以临时屏蔽内部程序的鼠标捕获，强制进行系统级别的原生文本选中：
- **Mac 环境** (iTerm2, macOS Terminal, Kitty 等): 按住 **`Option (⌥)`** 键 或 **`Fn`** 键不松手，再用鼠标拖拽选中文本。
- **Windows/Linux 环境** (Windows Terminal, GNOME Terminal, PuTTY 等): 按住 **`Shift`** 键不松手，再用鼠标拖拽选中文本。

### 2. 解决被困在历史记录无法回底的问题：一键退出
如果你因为鼠标向上滚动误入了 tmux 的浏览模式（键盘失效，按 End 没反应），按以下任意方式可瞬间解除冻结并回到最底部的输入框：
- **按 `q` 键**：最快捷推荐。tmux 中代表 Quit，一秒退出浏览模式并跳回底部。
- **按 `Esc` 或 `Ctrl+c`**：也能强行退出历史浏览状态。
- **用鼠标滚轮一直往下滚**：滚到最后一行（最新输出）时，tmux 会自动退出浏览模式，恢复正常交互。

### 3. 使用 tmux 原生复制模式 (Copy Mode)
完全脱离鼠标，使用纯键盘操作，不受任何程序劫持影响：
1. 唤醒 tmux：`Ctrl + b`
2. 进入复制模式：`[`
3. 使用方向键（或 h/j/k/l）移动光标到起始位置
4. 按 `空格键 (Space)` 开始标记
5. 移动光标选择区域
6. 按 `回车键 (Enter)` 复制并退出

### 4. 临时关闭 tmux 鼠标模式
如果需要大面积或高频次复制，可以临时解除劫持：
- 进入命令模式：`Ctrl + b` 然后按 `:`
- 关闭鼠标：输入 `set -g mouse off`
- （复制完后可通过 `set -g mouse on` 重新开启）

## Pitfalls
- 不要误以为是终端卡死或程序出 Bug。无法复制是终端模拟器在遵从“鼠标接管协议”；无法打字/回底是因为触发了 tmux 的滚动冻结机制。
- 不同操作系统的鼠标强制修饰键不同（Mac 常用 Option/Fn，Win/Linux 常用 Shift），容易记混导致无效操作。
- 即使不在 tmux 的冻结模式下，TUI 里试图用 Windows 常用的 `End` / `Ctrl+End` 跳转到底部也通常无效。跳回底部更推荐使用标准的 Vim 快捷键：**大写 `G`** (`Shift + g`)，或者随便打字触发输入框的自动跟随。
